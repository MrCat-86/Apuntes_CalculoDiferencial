## Introducción

Las **estructuras secuenciales** son las formas más simples de control de flujo en la programación. En una estructura secuencial, las instrucciones del programa se ejecutan de manera lineal, una tras otra, sin alteraciones en el orden de ejecución. Este tipo de flujo es el más común en los programas más sencillos y es esencial para comprender los fundamentos del control de flujo en programación.

En una estructura secuencial, las instrucciones son ejecutadas en el mismo orden en que están escritas en el código fuente.

## ¿Qué son las Estructuras Secuenciales?

En una **estructura secuencial**, las instrucciones se ejecutan una después de otra. No hay condiciones ni ciclos involucrados, solo la ejecución de una serie de pasos de manera directa y predecible. Este tipo de estructura es útil cuando no se requiere tomar decisiones o realizar repeticiones en el código.

### Características de las Estructuras Secuenciales:

- Son la forma más simple de control de flujo.
- Las instrucciones se ejecutan en el orden en que aparecen.
- No hay desviaciones o saltos en el flujo de ejecución.

### Ejemplo en Java de Estructura Secuencial:

```java
public class EjemploSecuencial {
    public static void main(String[] args) {
        // Declaración de variables
        int numero1 = 10;
        int numero2 = 20;
        int suma = numero1 + numero2;

        // Imprimir resultado
        System.out.println("La suma de " + numero1 + " y " + numero2 + " es: " + suma);
    }
}
```

En este ejemplo, las instrucciones se ejecutan en el orden en que se escriben:

1. Se declara la variable `numero1` con un valor de 10.
2. Se declara la variable `numero2` con un valor de 20.
3. Se realiza la suma de `numero1` y `numero2` y el resultado se guarda en la variable `suma`.
4. Se imprime el resultado de la suma.

Este es un flujo secuencial simple, ya que las instrucciones se ejecutan una tras otra sin ninguna condición ni bucles.

### Ejemplo con Entrada del Usuario:

Un programa que toma la entrada del usuario y realiza operaciones simples también utiliza una estructura secuencial. Aquí hay un ejemplo donde el programa solicita dos números al usuario y luego calcula su suma:

```java
import java.util.Scanner;

public class EjemploSecuencialEntrada {
    public static void main(String[] args) {
        // Crear un objeto Scanner para leer la entrada
        Scanner scanner = new Scanner(System.in);

        // Solicitar al usuario que ingrese dos números
        System.out.print("Ingrese el primer número: ");
        int numero1 = scanner.nextInt();

        System.out.print("Ingrese el segundo número: ");
        int numero2 = scanner.nextInt();

        // Calcular la suma
        int suma = numero1 + numero2;

        // Imprimir el resultado
        System.out.println("La suma de " + numero1 + " y " + numero2 + " es: " + suma);
        
        scanner.close();
    }
}
```

### Explicación del flujo:

1. El programa solicita al usuario que ingrese el primer número y lo guarda en la variable `numero1`.
2. Luego, el programa solicita el segundo número y lo guarda en la variable `numero2`.
3. A continuación, el programa realiza la suma de ambos números y guarda el resultado en la variable `suma`.
4. Finalmente, el programa imprime el resultado de la suma.

Al igual que el ejemplo anterior, el flujo de ejecución es secuencial, es decir, cada instrucción se ejecuta de manera ordenada, paso por paso.

## Ventajas de las Estructuras Secuenciales

- **Simplicidad**: Son fáciles de entender y escribir, lo que las hace ideales para programas sencillos.
- **Previsibilidad**: Al no haber condiciones ni ciclos, el flujo de ejecución es completamente predecible.
- **Base de estructuras más complejas**: Las estructuras secuenciales forman la base sobre la que se construyen otras estructuras de control de flujo, como las condicionales y los bucles.

## Conclusión

Las **estructuras secuenciales** son la forma más básica y directa de controlar el flujo de ejecución de un programa. Aunque son simples, son esenciales para la creación de programas más complejos y son la base sobre la que se agregan otros tipos de control de flujo, como las estructuras condicionales y de repetición. Comprender cómo funcionan las estructuras secuenciales es crucial para cualquier programador, ya que permiten ejecutar operaciones paso a paso de manera ordenada.
